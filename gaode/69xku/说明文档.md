# 69xku.com（69库）抓取说明

## 入口 URL

- 书籍页：`https://69xku.com/book/{书籍数字}/`  
  示例：[https://69xku.com/book/49851/](https://69xku.com/book/49851/)（[母女宠物饲养日记](https://69xku.com/book/49851/)）

## 章节目录

- **全书区**：`#list-chapterAll` 内 `dd > a`，`href` 形如 `/book/{书号}/{章节数字}.html`。
- **最新区**：同一页 `dl.book.chapterlist` 下 **直接子级** `dd > a`（不含 `#list-chapterAll` 内的条目），对应页面上「《书名》最新章节」下列出的若干条。
- **合并规则**：全书区按标题「第 n 章」升序；**仅出现在最新区**且不在全书区的链接排在 **manifest 末尾**（避免把最新更新列表插在全书开头）。`番外` 等排在数字章之后（与 `chapterNumberFromTitle` 约定一致）。

## 正文

- 优先 `#rtext`，其次 `.readcontent` 等；正文在 `#acontent` 内。
- 去掉常见站尾/域名推广行（如「本站永久域名」等）。

## 命令

```bash
node novel-workflow.js x69ku-49851 5
```

或：

```bash
node gaode/69xku/scrape-69xku.js "https://69xku.com/book/49851/" 5 --out-dir=novel-output/x69ku-49851 --merge --merge-title=母女宠物饲养日记
```

试跑可加数字限制章数；覆盖已抓分章：`--force` 或环境变量 `X69KU_FORCE=1`。
