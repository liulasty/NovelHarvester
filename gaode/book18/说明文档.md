# book18.org（`scrape-novel.js`）

与项目根目录通用流程一致：分章写入 `chapters/`，合并见根目录 `merge-novel.js`；`novel-workflow.js` 中 `scraper` 缺省为 `book18` 时使用本脚本。

## 目录页

- **推荐**：`/zh-hans/chapters/书名编码` 专章列表页，页面内 **`div.content-wrapper ul li`** 下每条均为章节链接。
- **兼容**：书籍页或阅读页侧栏目录；若 URL **不是** 专章列表路径，脚本会 **跳过第一条 `li`**（书籍总览）。

## 正文页

- 正文容器：**`.reader`**。
- 导航：`waitUntil: domcontentloaded`，超时 60s。

## 用法与环境变量

- 命令行传入目录页 URL；试跑前 N 章：URL 后加数字，如 `…/chapters/… 5`。
- **`--file`**：只从本地章节 URL 列表读取（默认项目根 `chapters_urls.txt`，可用 **`--url-file=`**）。
- **`BOOK18_CHAPTERS_URL`**：未传 CLI 目录 URL 时的默认章节目录页。
- 其他：`--out-dir`、`--merge`、`--merge-title=`，与根目录说明一致。

## 配置示例

`novel-targets.json` 中未写 `scraper` 或显式 `book18` 的目标（如 `xnl`、`xiaohua-xyz`）使用本脚本。
