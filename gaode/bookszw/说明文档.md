# bookszw.com 抓取说明

## 入口 URL

- 书籍目录：`http://www.bookszw.com/{分类数字}/{书籍数字}/`  
  示例：[http://www.bookszw.com/22/22313/](http://www.bookszw.com/22/22313/)  
- 脚本会将 `bookszw.com` 的 **https** 规范为 **http**（该站 TLS 可能不可用）。
- 亦支持带 `index.html`、`index_2.html` 等分页的目录地址。

## 章节目录

- 在 **`#list`** 内按 **`<dt>` / `<dd>`** 分段：标题匹配「最新章节」「最近更新」等的一整段归入 **最新区**；其余 `<dd>` 归入 **正文区**（含「正文」「章节列表」等 dt）。
- **本站常见版式**（与笔趣阁类似）：每一页目录里 **`#list` 中按出现顺序** 往往是 **前约 10 条为「最新章节」**，**后面至多约 20 条为本页正文目录**；若页面未用可识别的 `<dt>` 分隔，脚本在 **最新区解析结果为空** 且 **`#list` 内有效章节数大于 `BOOKSZW_LATEST_TOP`** 时，会按该顺序把 **前 `BOOKSZW_LATEST_TOP` 条（默认 10）** 当作最新区，其余当作本页正文区。可用环境变量 **`BOOKSZW_LATEST_TOP`** 覆盖条数；设为 **`0`** 可关闭此回退（仅信 `<dt>` 分段）。
- 仅扫 `href` 为 `/{分类}/{书籍}/{数字}.html` 的章节链接（不含 `_1` 等分页后缀）。
- **合并规则**：正文区链接按标题中的 **「第 n 章」** 升序排序；**仅出现在最新区**、且不在正文区的链接排在 **manifest 末尾**（同样按章号升序），避免把「最新十章」插在全书开头。
- 同一 `href` 若既有「开始阅读」又有「第1章」等文案，保留含 **「第 n 章」** 的标题。
- 目录多页时跟随 **「下一页」**，各页的 main/latest 会先合并再排序。

## 正文

- 依次尝试：`#content`、`#contents`、`#chaptercontent`、`.ReadAjax_content`、`.showtxt`、`#BookText`（取到正文长度大于约 80 字即采用）。
- **章内分页**：在 `7263101.html` 之后继续请求 `7263101_1.html`、`7263101_2.html`… 直到 `h1` 或页面标题中出现 **「第 M/N 页」且 M≥N**，或正文过短/与上一屏重复。多段正文以空行拼接。若正文内有「本章未完」等续页提示，**在去广告行之前**即参与判断是否拉下一屏（否则提示行被删掉后会误判为单页）。
- 去掉常见站尾提示行（如「请记住本书首发域名」等）。

## Cloudflare / 人机验证

站点常返回 **403 + Cf-Mitigated: challenge** 或 Turnstile 页面，**默认无头 Chromium 往往无法通过**。

可选做法：

1. **有头浏览器**（推荐首次验证）：在运行前设置环境变量  
   `NOVEL_HEADLESS=0` 或 `BOOKSZW_HEADED=1`  
   使用本机已安装的 **Chrome**（`channel: 'chrome'`）打开窗口，在页面内完成验证后再继续。
2. **保存会话**：验证通过后可用 Playwright 导出 `storageState` JSON，之后设置  
   `BOOKSZW_STORAGE_STATE=路径\bookszw-state.json`  
   再跑脚本可复用 Cookie（路径可为项目根下相对路径）。
3. 延长等待：`BOOKSZW_CHALLENGE_TIMEOUT_MS`（默认 **120000**，即 120 秒；等待期间控制台会每约 8 秒打印一行进度，避免误以为卡死）。

## 命令

```bash
set NOVEL_HEADLESS=0
node novel-workflow.js bookszw-22313 3
```

或：

```bash
node gaode/bookszw/scrape-bookszw.js "http://www.bookszw.com/22/22313/" 3 --out-dir=novel-output/bookszw-22-22313 --merge --merge-title=九阳帝尊
```
