# 9ksw.com（九库书屋）

## 入口 URL

书籍章节列表页：`https://9ksw.com/novel{书籍id}/`

示例：`https://9ksw.com/novel45382/`（[仙子破道曲 目录](https://9ksw.com/novel45382/)）

## 目录结构

| 区域 | 选择器 | 说明 |
|------|--------|------|
| 最新章节 | `#new-chapter ul.list-chapter a` | 每页约 6 条，与全书列表重叠；合并时用于补全标题 |
| 全书目录 | `#list-chapter ul.list-chapter a` | 每页约 50 条正文章节链接 |
| 分页 | `#pagination a[href*="?p="]` | 如 `?p=2` … `?p=4`，脚本读取最大 `p` 后逐页抓取 |

章节链接路径：`/novel{id}/chapter{N}.html`（`N` 为从 0 起的序号，与「第几章」无必然相等关系）。

合并顺序：按第 1～末页的 **DOM 顺序** 拼接全书区（保证「间章」等插在站内原位置），再用「最新章节」区覆盖同链接的标题，并把仅出现在最新区、不在全书区的链接接在末尾。

## 正文

阅读页 `#chapter-content` 初始为「加载中」，由页面脚本 POST `/conapi.php` 拉取正文后写入。脚本会等待该请求或轮询至正文长度足够且不含加载提示。

## 命令示例

```bash
node gaode/9ksw/scrape-9ksw.js "https://9ksw.com/novel45382/" 5 --out-dir=novel-output/9ksw-45382 --merge --merge-title=仙子破道曲
node novel-workflow.js 9ksw-45382 3
```

环境变量：

| 变量 | 说明 |
|------|------|
| `N9KSW_CHAPTERS_URL` | 目录入口 URL |
| `N9KSW_FORCE=1` 或 `--force` | 强制覆盖已有分章 |
| `NOVEL_HEADLESS=0` | 有头浏览器（有时更不易被拦） |
| `N9KSW_DELAY_MS` / `N9KSW_DELAY_JITTER_MS` | 每章前间隔 + 随机抖动，减轻限流（默认 1200 + 0～800ms） |
| `N9KSW_COOLDOWN_MS` | 单章失败后冷却再抓下一章（默认 5000） |
| `N9KSW_CHAPTER_RETRIES` / `N9KSW_GOTO_RETRIES` | 正文整章重试、单次 `goto` 重试（默认 3 / 4） |
| `N9KSW_CONTENT_WAIT_MS` / `N9KSW_API_WAIT_MS` | 等正文就绪、等 `conapi.php` 的最长时间（默认 120000 / 60000） |
| `N9KSW_USER_AGENT` | 自定义 UA（可选） |

断线、超时、被关连接时可先加大间隔与冷却，或改用有头模式分多段跑（已成功的章会跳过）。
