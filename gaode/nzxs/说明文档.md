# nzxs.cc 抓取说明

## 入口 URL

- 书籍目录页：`https://www.nzxs.cc/book/{书籍数字id}/`
- 也支持 `/book/{id}/n/`（脚本会归一化到 `/book/{id}/` 后再发现章节）。

## 章节目录（多页）

1. 在 `#novel_info` 内找到标题为 **章节列表** 的 `h3.wrap-title`，取其下方 `ul.chapter-list` 中的 `/read/{id}/{章节文件id}.html` 链接。
2. 若首页存在 **阅读更多章节**（`a.btn-mulu`，文案亦可能为「查看更多章节」等；`href` 形如 `/book/{id}/1/`），再打开该 URL，解析页面上 **链接数最多** 的 `ul.chapter-list`（第二页模板可能没有 `#novel_info`）。
3. 两页结果按顺序合并，**同一 `href` 只保留第一次出现的标题**。

说明：站点按钮文案以「阅读更多章节」为主；脚本同时匹配「查看更多章节」与「章节目录」等，避免改版后失效。

## 正文

- 主容器：`#txt`（由页面脚本写入段落，需等待有字后再取）。
- **章内分页**：首屏为 `…/84692256.html`，后续为 `…/84692256_1.html`、`_2.html`…；以 `h1` 中的 `（第N页）` 连续递增为准，若出现回到 `（第1页）` 或页码不连续则停止。
- 会去掉常见站尾广告行（如「一秒记住新域名」「请勿开启浏览器阅读模式」）。

## 命令

```bash
node gaode/nzxs/scrape-nzxs.js "https://www.nzxs.cc/book/352626/" 5 --out-dir=novel-output/nzxs-352626 --merge --merge-title=仙姝墮
```

或：

```bash
node novel-workflow.js nzxs-xsz 5
```
