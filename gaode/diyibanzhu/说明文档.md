# diyibanzhu.quest 第一版主网（`scrape-diyibanzhu.js`）

与 book18 流程一致：分章进 `chapters/`，合并进 `merged/全文合并.txt`。`novel-workflow.js` 中 **`scraper`: `diyibanzhu`** 时调用本脚本。

## 目录页

- 书籍章节目录为 **`/list/{书籍id}.html`**。
- 页内通常有「最新章节」与 **「章节列表」** 两块（`h4` 文案含「章节列表」）。脚本只在 **该标题所在 `.mod` 块** 下读取 **`ul.list`** 中的 `/view/…` 链接，避免把「最新章节」或页脚推荐混入。
- **跳过** `…_start.html`（从头阅读）。

## 目录分页

- 章多时底部有「下页」，链接形如 **`/list/{id}_34_2.html`**。
- 脚本按 pathname 中的书籍 id 识别 **指向同书目录的「下页」**，依次合并多页目录中的章节；完整列表写入书籍根目录 **`chapters_manifest.json`**。

## 正文页

- 正文在阅读页 **`#nr1`**（位于 `#ChapterView .page-content`），与站点 PC 版一致。
- 不依赖书文 `.wen` 或 TImg。

## 环境变量（可选）

- **`DIYIBANZHU_CHAPTERS_URL`**：未传 CLI 目录 URL 时作为默认目录页。

## 配置示例

见项目根 `novel-targets.json` 中 id **`diyibanzhu-xzduo`**。
