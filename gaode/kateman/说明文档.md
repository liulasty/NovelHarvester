# kateman.net（书海阁小说网）

## 概述

- **域名**: www.kateman.net
- **主题**: 玄幻/都市小说
- **特征**: 章节正文使用 AES-CBC 加密（CryptoJS），浏览器端解密

## 目录页

- URL 模式: `https://www.kateman.net/shuwu/{bookId}/`
- 章节链接选择器: `.panel.panel-default` 内 `a[href*="xiaoshuo_"]`
- 页面默认按倒序排列（新章在前），脚本取 link 数量最多的 panel，保持页内顺序
- **注意**：网站连接不稳定，偶发 `ERR_CONNECTION_CLOSED`，脚本内部有 3 次重试 + commit 降级；由于 `waitUntil: 'domcontentloaded'` 可能超时，若重试耗尽会回退至 `commit` + 手动等待

## 正文页

- URL 模式: `https://www.kateman.net/xiaoshuo_{bookId}/{chapterId}.html`
- 正文容器: `#booktxthtml`（内含 `<p>` 标签段落）
- 正文通过内联脚本调用 `$('#booktxthtml').html(x(encrypted, key, iv))` 解密
- **解密参数**:
  - 算法: AES-CBC
  - Key: `"encryptedDatastr"`（UTF8）
  - IV: Base64 字符串
  - Padding: ZeroPadding
- 脚本使用 `waitUntil: 'commit'`（避免因慢连接超时），等 `#booktxthtml` 出现后等待 `CryptoJS` / `x` 函数就绪，再 `eval()` 解密脚本

## 章内分页

- 同章分页 URL: `{chapterId}_2.html`、`{chapterId}_3.html` …
- 判断: 页面中存在文本为「下一页」的链接
- 脚本会循环跟随直到无「下一页」或达到 `MAX_PAGES_PER_CHAPTER`（20）上限

## Scraper 键

`kateman`（在 `lib/orchestrator/registry.js` 中注册）

## 命令示例

```bash
# 完整抓取并合并
node novel-workflow.js kateman-hqqhpm

# 试跑前 5 章
node novel-workflow.js kateman-hqqhpm 5

# 直接调用脚本
node gaode/kateman/scrape-kateman.js https://www.kateman.net/shuwu/hqqhpm/ --out-dir=novel-output/kateman-hqqhpm --merge --merge-title=家里的美人犬老婆
```

## 注意事项

- **连接不稳定**：该站 SSL 重协商较慢，且偶发连接断开；脚本已内置多重重试，但仍可能需要较长时间完成全本抓取
- **无 Cloudflare**：本站未使用 Cloudflare 人机验证，无头模式可直接运行
- **依赖**：脚本需要 `Playwright` 浏览器执行 JavaScript 解密，无法通过裸 HTTP 请求获取纯文本
- **章顺序**：目测首次拉取的 panel 为页面靠后的「按序排列」区块；若发现乱序，可手动检查 `chapters_manifest.json` 调整
